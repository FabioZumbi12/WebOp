WebOp Permissions Handler

If a player has been opped, they are universally entitled to every permission, regardless of whether or not they are on any of these lists. If you are opped, you do not need to be on any of these lists.

In the interest of security, only opped users are able to modify these lists.

Server Whitelist

This whitelist determines which players are allowed to login to the gameserver whilst whitelist mode is enabled.

Whitelist mode is currently: {serverWhitelistBoolean}


    {serverWhitelist_users}

WebOp Login

This whitelist determines which players are entitled to login to the WebOp service. Users that are not on this list are refused access to all functionality on this website. If a user is removed from this list and is already logged in, their session becomes invalid and are thus unable to use the WebOp service.

    {accessWhitelist_users}


Add Players:

Console View

This whitelist determines which players are entitled to view the console in the WebOp service. Users are additionally allowed to execute commands as themselves only (just like 'in-game'), and are denied the ability to execute commands as the ConsoleCommandSender.

    {consoleViewWhitelist_users}


Add Players:

Console As OP

This whitelist determines which players are entitled to issue commands in the WebOp console as the ConsoleCommandSender.
**WARNING** This entitlement effectively gives the player OP permissions and will be able to issue ANY CONSOLE COMMAND from the WebOp console.

    {consoleAsOpWhitelist_users}
